Exercise 4 - Geometric transformations 2

Learning objectives
  • Compute transformations based on input points.
  • Manipulate objects using mouse input.
-We will be using the library glm to facilitate the manipulation of matrices and vectors. The tutorial https://learnopengl.com/Getting-started/Transformations might prove helpful. You also have the option to use the glmutils library, which is included in the exercise.
Exercise 4.1
 Throwing paper planes
  • When a left mouse button click is detected, move the plane to the clicked position.
  • When you click and hold the left mouse button, rotate the plane to align it with the line defined by the start click position and the vector from that point to the current mouse pointer position.
  • Once you release the left mouse button, launch the plane in the direction of the line from the start click position to the end click position. Set the flight speed to be proportional to the length of the line
  • The plane should neither move nor rotate if the start click position and current/end position are the same (not enough information to define a line).
Exercise 4.2
Graphical interface feedback
  • Working on the output of 4.1:
  • Initialize and draw a SceneObject using the arrow geometry (defined in the primitives.h file).
  • The arrow should only be rendered while the left mouse button is pressed.
  • Set the position and rotation of the arrow to align it with the line defined by the start click position and the vector from that point to the current mouse pointer position.
  • Set the scale of the arrow so that the tip is located under the mouse pointer position.
Exercise 4.3
Trackball rotation interface
  • The virtual trackball slides present equations to:
    • Project an image plane coordinate (x,y,0) on a radial 3D surface;
    • Map two positions in that surface into an axis angle rotation representation (the actual matrix can be obtained using glm::rotate(angle, rotation axis)).
  • Implement the Shoemake trackball.
  • Implement the Anderson trackball.
