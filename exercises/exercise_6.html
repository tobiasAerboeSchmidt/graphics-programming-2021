Exercise 6.1
 Scan conversion of triangles:

Examine the implementation of the main.cpp file. You will see that our openGL application looks different. We have no geometry, no VAOs or VBOs, and we do not set the depth parameters nor clear the image and depth buffers.
Instead, we are writing the colors to a memory buffer that store the colors (CustomFrameBuffer class), using it as an OpenGL texture, and copying it to the window framebuffer (see the blit function in the main loop). So, we are setting the colors of the pixels directly in the C++ application, instead of using openGL's rendering pipeline.
The objective is to distance ourselves from OpenGL, so that we can understand what goes into the implementation of the fixed (nonprogrammable) stages of the rendering pipeline. Today we start with rasterization:
  • Checkout the implementation of the CustomFrameBuffer class, to have an idea of how it works and how we can use it.
  • Checkout the implementation of the class for scan-line conversion of lines (linerasterizer.h/.cpp). Do you identify the algorithms that you have seen in class?
  • Finally, based on the material in the slides, implement the rasterization algorithm for triangles. This will require updating the classes edgerasterizer and trianglerasterizer, where it says TODO. You will notice that the slides already present you with the code, instead of requiring you to create your own implementation, so this is an opportunity to read and understand what that code is doing and how it relates to the math in the slides.
  • Once complete, the triangle rasterizer should fill the triangle like in the image in the left (toggle lines and triangle fill rendering by pressing 1 and 2, respectivelly).
